PXE Booting via AWS

Bare-Metal device initiates PXE rebuild. Bare-Metal device connects to EC2 Instance which is hosting a DHCP service as well as a TFTP service. DHCP service returns a DHCP IP and the location of the TFTP Service Bare-Metal device makes a TFTP request for the boot files. This request invokes an event-triggered Lambda function that will generate a Pre-Signed URL from the Current Build Files S3 Bucket. The Pre-Signed URL is sent to the Bare-Metal device via the TFTP service. Three important buckets within this process, the Bucket for updated Boot Files, and current Boot Files. S3 Bucket containing the updated boot files (new updates monthly) S3 Bucket containing the currently in use boot files S3 Bucket containing last patch (Recovery) S3 Bucket containing the currently in use boot files gets updated by a time triggered Lambda function that replicates the files from the updated Boot Files bucket into the currently in use Boot Files bucket Prior to updating boot files to new patch/update, old files from the current boot files bucket get replicated into the recovery bucket.
